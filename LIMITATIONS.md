# Limitations

To be completely transparent about the current state and intended scope of CENTINELA:

1. **Synthetic Data Only**: All customer data, transactions, and evidence are synthetically generated for demonstration purposes. No real PII or financial data is used.
2. **Mock Banking APIs Planned**: The interactions with core banking systems and receiver banks will be simulated using mock APIs, not real banking integrations.
3. **Not Legal Advice**: The workflows, decisions, and logic represented in this project do not constitute legal or compliance advice for fraud handling.
4. **Not Production Banking Infrastructure**: This is a hackathon prototype designed to demonstrate UiPath Maestro Case capabilities. It lacks the security, scale, and compliance rigor of real production banking systems.
5. **Not Connected to Real Bank Systems**: There is zero connectivity to any actual financial institution.
6. **UiPath Labs Connection Pending**: The current codebase represents Step 1 (the foundation and evidence generation). Integration with UiPath Labs and Maestro Case is planned but pending.
