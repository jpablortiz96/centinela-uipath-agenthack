# UiPath Evidence Pack

## A. Executive Summary
UiPath Maestro Case provides the governed case-management and orchestration layer for CENTINELA, handling states, tasks, and team routing. The **CENTINELA Runtime API** performs the heavy lifting of deterministic fraud investigation, mock banking API integrations, retry handling, policy evaluation, audit export generation, and analyst intelligence visualization. This architecture ensures high-level orchestration is managed by UiPath while complex domain logic runs robustly in the runtime.

## B. What runs in UiPath
*   **Maestro Case stages:** Intake, Evidence Review, Investigation, Human Decision, Resolution, and Audit Export.
*   **Human tasks & routing:** Forms and queues to involve fraud operations analysts.
*   **SLAs:** Case deadlines and expected turnarounds.
*   **Auto-generated routing rules:** Orchestration flow logic.
*   **Integration Service Connector Activity:** Maestro calls the CENTINELA Runtime API directly through Integration Services to run investigations and apply decisions.
*   **Connected debug execution:** The end-to-end flow is fully executable in Studio Web debug mode.
*   **Published Maestro Case v1.0.0:** The baseline orchestration definition is published.

## C. What runs in CENTINELA Runtime
*   **Public Runtime API:** Hosted on Render, serves as the target for UiPath.
*   **Retry policy:** Handles temporary failures in receiver bank APIs with backoff and exhaustion mechanics.
*   **Fraud policy engine:** Evaluates rules (e.g., risk levels, conflict status) to determine if a case can be auto-resolved or requires human intervention.
*   **SLA status:** Deterministic turnaround calculations.
*   **Human gate:** Enforces human review for high-risk or retry-exhausted cases.
*   **Audit export v2:** Packages the complete immutable case state (timeline, risk, decisions) for UiPath to fetch at closure.
*   **Fraud Intelligence Layer:** Generates deterministic network graphs, priority scores, decision simulations, evidence checklists, and linked signals.
*   **Analyst Console & Judge Replay:** Operational interfaces to visualize the intelligence data.

## F. Connected Publish Limitation
While the published Maestro Case v1.0.0 exists and connected debug works flawlessly, publishing the *connected version* of the project is currently blocked by a UiPath Labs custom connector packaging/export issue (`elements unknown` error during download). This is documented transparently in `docs/UIPATH_PRODUCT_FEEDBACK.md`. 

## G. Judge Interpretation
*   **This is not a production banking system.** It is a hackathon prototype.
*   **It uses deterministic synthetic adapters.** No real banking data is used.
*   **It demonstrates enterprise-ready orchestration:** The project emphasizes human accountability, retry handling, auditability, and explainable decision support over uncontrolled probabilistic generation.
